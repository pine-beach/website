0.1 ‚Äî Verify OpenClaw is installed

openclaw --version

You should see a version number like openclaw/2026.x.x. If you see command not found, go to 0.2 below.

0.2 ‚Äî Install OpenClaw (if needed)

macOS or Linux:

curl -fsSL https://openclaw.ai/install.sh | bash

This downloads the CLI, installs it globally, and launches an onboarding wizard. Follow the wizard ‚Äî it will ask you to configure your AI provider.

Windows: OpenClaw requires WSL2. First install it (wsl --install in PowerShell as Administrator, restart), then open your WSL2 terminal and run the Linux command above.

‚ö†Ô∏è Common issue ‚Äî command not found after install: Run npm prefix -g to find where npm installs global packages, then add that bin folder to your PATH.

0.3 ‚Äî Verify your AI model is configured

openclaw models status

You should see at least one provider listed as active. If nothing is configured, run:

openclaw onboard

The wizard will ask you to choose between:

  • Anthropic API key ‚Äî best for most users. Get one at console.anthropic.com ‚Üí API Keys ‚Üí Create Key. Looks like sk-ant-.... Costs are usage-based (typically a few dollars/month for a small team).
  • Claude subscription token ‚Äî uses your existing Claude subscription. Run claude setup-token on a machine with the Claude CLI installed, then paste the output when prompted.

0.4 ‚Äî Verify the Gateway is running

openclaw gateway status

Should say running or active. If not:

openclaw gateway start

The setup script does the heavy lifting: it asks you a series of questions, then generates your entire workspace ‚Äî all files pre-filled with your company name and agent names, ready to use.

2.1 ‚Äî Run the script

Make sure you're inside the unzipped folder, then run:

bash scripts/setup.sh ~/workspace

~/workspace is where your agent files will live. Change it to any path you like. The rest of this guide assumes ~/workspace.

2.2 ‚Äî What the script will ask

The script runs in 4 steps:

1/4
Company & owner
Your company name and what your agents should call you (first name). Leave company name blank to use "My Company" as default.
2/4
Choose your agents
CEO is always included. Press Enter (or Y) for each role you want, N to skip. Pick what matches your immediate needs ‚Äî adding more later takes 5 minutes.
3/4
Name your agents
Give each agent a name, or press Enter for the defaults. These are personality names, not role names ‚Äî your CEO can be "Marcus" or "Aria" or anything you like.
4/4
Workspace generation
The script builds everything and prints a config snippet at the end. Takes about 10 seconds. Don't interrupt it.

3A.1 ‚Äî Open your config file

OpenClaw reads its config from ~/.openclaw/openclaw.json (JSON5 format ‚Äî allows comments and trailing commas). Open it:

nano ~/.openclaw/openclaw.json

3A.2 ‚Äî Add your agents

Replace the contents with this. Change every path that says /home/yourname/workspace to your actual workspace path (run echo ~/workspace to find it). Remove blocks for roles you didn't select in Phase 2.

// ~/.openclaw/openclaw.json { agents: { defaults: { model: { primary: "anthropic/claude-sonnet-4-5", }, }, list: [ { id: "ceo", workspace: "/home/yourname/workspace/agents/ceo", model: "anthropic/claude-opus-4-6", // CEO gets Opus ‚Äî worth it }, { id: "cto", workspace: "/home/yourname/workspace/agents/cto", }, { id: "cmo", workspace: "/home/yourname/workspace/agents/cmo", }, // ... add/remove roles to match your Phase 2 selections ], }, }

3A.3 ‚Äî Copy credentials to each agent

Each agent has its own auth store. Copy your main credentials across:

# Create auth directories for each agent mkdir -p ~/.openclaw/agents/ceo/agent mkdir -p ~/.openclaw/agents/cto/agent # (repeat for each agent you created) # Copy credentials cp ~/.openclaw/agents/main/agent/auth-profiles.json ~/.openclaw/agents/ceo/agent/ cp ~/.openclaw/agents/main/agent/auth-profiles.json ~/.openclaw/agents/cto/agent/ # (repeat for each agent)

‚ö†Ô∏è Can't find auth-profiles.json? Run find ~/.openclaw/agents -name "auth-profiles.json" to locate it, then copy from that path.

3A.4 ‚Äî Reload and verify

openclaw gateway restart openclaw agents list

You should see each agent listed. If one is missing, there's usually a typo in the config path. Run openclaw doctor for diagnostics.

Phase 3A checkpoint

Option 1 ‚Äî Upgrade to CEO

  1. Find your existing workspace: openclaw agents list
  2. Copy the CEO SOUL: cp ~/workspace/agents/ceo/SOUL.md /path/to/your/workspace/SOUL.md
  3. Copy company constitution: cp ~/workspace/AGENTS.md /path/to/your/workspace/AGENTS.md
  4. Edit ~/.openclaw/openclaw.json ‚Äî add an agents.list, make your existing workspace the CEO entry (id: "ceo", default: true), then add the new agent blocks
  5. Copy credentials to new agents (see Phase 3A.3)
  6. Restart: openclaw gateway restart && openclaw agents list

Option 2 ‚Äî Keep existing, add alongside

Open ~/.openclaw/openclaw.json and add the Business in a Box agents to your existing list. If you're in Pattern A (no agents.list), make your existing agent explicit first:

{ agents: { defaults: { model: { primary: "anthropic/claude-sonnet-4-5" }, }, list: [ // YOUR EXISTING AGENT ‚Äî unchanged { id: "main", default: true, workspace: "/home/yourname/.openclaw/workspace", // do not change }, // BUSINESS IN A BOX ‚Äî new additions { id: "ceo", workspace: "/home/yourname/workspace/agents/ceo", model: "anthropic/claude-opus-4-6", }, { id: "cto", workspace: "/home/yourname/workspace/agents/cto" }, // ... add roles you selected in Phase 2 ], }, // Keep all your other existing config ‚Äî do NOT delete anything }

Then validate, copy credentials, and reload:

openclaw doctor # fix any errors, then: openclaw gateway restart openclaw agents list

Can I skip Discord? Yes. Talk to agents via openclaw dashboard in your browser, or via the CLI: openclaw agent --agent ceo --message "your message". Discord is the designed experience, but not required.

4.1
Create a Discord server
In Discord, click + in the sidebar ‚Üí Create My Own ‚Üí For me and my friends. Name it something like "[Your Company] HQ".
4.2
Create channels for each agent
One text channel per agent: #ceo-office, #dev-workshop, #marketing-hub, #finance-desk, #security-ops, #support-desk, #general, #war-room.
4.3
Create a Discord bot
Go to discord.com/developers/applications ‚Üí New Application ‚Üí name it after your CEO. Under Bot, enable Message Content Intent (required) and Server Members Intent. Click Reset Token and copy it.
4.4
Enable Developer Mode & collect IDs
User Settings ‚Üí Advanced ‚Üí Developer Mode ON. Then: right-click server icon ‚Üí Copy Server ID. Right-click your username ‚Üí Copy User ID. Save both.
4.5‚Äì4.6
Add bot token to OpenClaw
# Never paste the token in a chat message openclaw config set channels.discord.token '"YOUR_BOT_TOKEN"' --json openclaw config set channels.discord.enabled true --json openclaw gateway restart
4.7
Configure server in OpenClaw
Add the Discord config block to ~/.openclaw/openclaw.json:
channels: { discord: { enabled: true, token: "YOUR_BOT_TOKEN", groupPolicy: "allowlist", guilds: { "YOUR_SERVER_ID": { requireMention: false, users: ["YOUR_USER_ID"], }, }, }, },
4.8
Invite the bot to your server
In the Developer Portal ‚Üí OAuth2 ‚Üí URL Generator: check bot and applications.commands. Bot permissions: View Channels, Send Messages, Read Message History, Embed Links, Attach Files. Copy the URL, paste in browser, select your server, Authorize.
4.9
Pair with your bot
DM your bot in Discord. It will send a pairing code. Then:
openclaw pairing list discord openclaw pairing approve discord CODE

IssueFix
